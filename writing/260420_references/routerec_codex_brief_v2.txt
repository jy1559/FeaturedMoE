--------------------------------------------------------------------------------
 옵션 A. Dataset Heterogeneity Score  ★★★★★ (무조건 추천)
--------------------------------------------------------------------------------
 목적: 저자 ML-1M/Beauty 방어의 최대 약점("preference-stable 직접 증거
       없음")을 정량화. 새 모델 훈련 없음.
       + PART 2의 REF-ML-1, REF-ML-2 external citation과 결합되면 방어력
         거의 완성.
 방법: 각 dataset의 train set에서 3개 지표 계산:
       - Mean intra-session category entropy  (Focus 이질성)
       - Mean inter-session repeat rate       (Memory 안정성)
       - Coefficient of variation of inter-event time  (Tempo 변동성)
 산출물: Appendix에 dataset × metric 표 1개 + 본문 1~2문장 추가.
 기대 효과:
  - ML-1M, Beauty가 "low heterogeneity" 쪽에 오고, KuaiRec, Foursquare가
    "high heterogeneity"에 오면 저자 가설을 정량 뒷받침.
  - 본문 Q1 섹션 직후에 한 문장으로 링크 가능:
      "Appendix X reports three dataset-level heterogeneity indicators
       that rank-correlate with RouteRec's relative gain, supporting
       the hypothesis that behavior-guided routing provides the largest
       leverage where behavioral regimes actually differ."
 소요 시간: 1~2시간 (데이터 통계만).
 Codex 판단 포인트: 지표 정의는 cue family(Tempo/Focus/Memory)와
   구조적으로 일치해야 함. Tempo CoV 대신 inter-event gap std를 쓸지
   결정은 Codex가 데이터 분포 보고 판단.

--------------------------------------------------------------------------------
 옵션 B. Random-Feature Sanity Check  ★★★★
--------------------------------------------------------------------------------
 목적: "behavior cue가 정말 routing을 drive하는가"의 가장 강력한 sanity
       check. 리뷰어 공격점 차단.
 방법: behavior cue를 같은 차원의 random noise로 교체한 variant를 1개
       dataset(KuaiRec 추천 - RouteRec 가장 강한 곳)에서 학습.
 산출물: 표 1개 (RouteRec real / RouteRec random / SASRec) + 본문 Q2
        한 문장.
 기대 효과: random feature가 성능 drop이면 cue의 필요성 직접 입증.
 소요 시간: 2~4시간 (1개 모델 추가 학습).
 Codex 판단 포인트: 만약 random feature variant가 예상만큼 안 떨어지면
   논문 메시지가 약해지는 역효과. 돌려서 결과 확인 후 넣을지 결정.

--------------------------------------------------------------------------------
 옵션 C. Seed Variance (Significance)  ★★★★
--------------------------------------------------------------------------------
 목적: 모델들이 공통 지적한 "seed variance 없음" 방어. ML-1M 같은 좁은
       margin에서 "noise 아니냐" 공격 차단.
 방법: 2개 dataset(1 dynamic + 1 stable, 예: KuaiRec + ML-1M) × RouteRec
       + strongest baseline 1개 × 3 seeds. Total 12 runs. 병렬로 돌리면
       overnight 가능.
 산출물: Table 1 footnote 또는 Appendix:
         "On KuaiRec and ML-1M, we additionally ran 3 seeds for RouteRec
          and the strongest baseline; std < 0.003 across all reported
          metrics."
 소요 시간: overnight 병렬.
 Codex 판단 포인트: 12 runs가 시간 내 끝나는지 확인. 안 되면 2 seeds로
   축소.

--------------------------------------------------------------------------------
 옵션 D. Routing-Cue Correlation Diagnostic  ★★★
--------------------------------------------------------------------------------
 목적: Q5의 "behavior cue → routing" 주장을 aggregate 지표로 보강.
       현재 Figure 5 case 3개 외에 거시적 증거 없음.
 방법: 이미 학습된 RouteRec 모델 inference log에서:
       - Stage별로 (behavior cue value, routing probability)의
         Spearman correlation 계산
       - Cue family × group의 대각 성분이 off-diagonal보다 높은지 확인
 산출물: 4x4 correlation 표 (Appendix H 확장) + 본문 Q5 한 문장.
 소요 시간: 1~2시간 (기존 모델 log 재활용).
 Codex 판단 포인트: 대각 성분이 예상보다 약하면 오히려 증거가 약해질
   위험. 계산 후 결과 보고 반영 결정.
