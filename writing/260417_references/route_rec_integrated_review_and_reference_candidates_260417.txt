1. 현재 버전 총평
==================================================

이번 버전은 이전보다 분명히 좋아졌다.
특히
- introduction의 3 challenge
- methodology의 3 design axis
- appendix의 fixed configuration / reproducibility 표
가 한 축으로 정렬되기 시작했다.

지금은 적어도 reviewer가
"이 논문이 뭘 하려는지 모르겠다"
라고 하기는 어려운 상태다.

핵심 메시지는 비교적 명확하다.

- macro / mid / micro의 coarse-to-fine stage로 나누고,
- hidden-state-only가 아니라 behavior-guided routing으로 expert allocation을 한다.

즉 concept-level story는 이제 잘 보인다.

다만 아직도 reviewer가 꼬치꼬치 보면 걸 수 있는 부분은 남아 있다.

(3) sampled subset 사용 사실이 main table에서 바로 보이지 않음

- 방법:
  1) Table 3를 A.2에 흡수하거나,
  2) heading을 삭제하거나,
  3) 최소한 2~3문장으로 역할을 적기.

[우선순위 3] Appendix D의 "Table ??" 제거
- placeholder는 제출본 신뢰도를 크게 깎는다.
- 표를 넣지 못하면 subsection 자체를 잠시 내리거나, 최소한 placeholder는 삭제해야 한다.

[우선순위 4] Table 3를 exact selected configuration 중심으로 바꾸기
현재 Table 3 일부는 search-space/track 정보를 주고 있다.
예:
- Experts per group: C ∈ {2,3,4}
- Router hidden width: representative range 48–128

최소한 아래는 최종값으로 고정하는 게 좋다.
- selected C
- total E
- selected router hidden width
- lambda_cons, lambda_z
- optimizer / learning rate / weight decay / dropout
- RouteRec best candidate run이 어떤 exact config였는지

macro/mid는 sequence-wise route를 broadcast한다고 이미 써놨기 때문에,

추천 문장:

[우선순위 7] category / theme proxy 설명 한 문장 추가

reader는 category 없는 dataset에서 theme proxy가 뭔지 궁금해할 수 있다.

4. methodology와 appendix를 더 설득력 있게 만드는 방법
==================================================

[4-1. framework vs instantiation 구분을 더 분명히]
현재는 3.1에서 한 번 선언했지만, 이후 본문에서 "fixed paper instantiation"이 계속 반복된다.
이 자체가 틀린 건 아니지만, 너무 자주 나오면 framework paper와 selected config report가 섞여 보인다.

추천 방식:
- 3.1 끝에서 한 번 강하게 구분
- main text는 개념과 이유 중심
- exact values / selected hyperparameters / practical setting은 Appendix A와 Table 2–3로 모으기

즉,
main text = why / what / design
appendix = exact selected setting

[4-2. 3.4의 plain-English bridge 강화]
Equation (6)–(8) 직전 혹은 직후에 아래 요약 문장을 직접 넣으면 좋다.

추천:
"In plain terms, RouteRec first selects the most relevant behavioral control view, and then allocates capacity within that view."

혹은
"The router first chooses the family-level control view, then chooses the expert within that view."

이 문장은 non-expert reviewer에게 매우 유용하다.

[4-3. dense mixture 채택 이유를 지금보다 더 적극적으로]
현재는 dense mixing을 선택했다고 잘 적었지만,
조금 더 이번 논문의 목적과 연결하면 좋다.

추천 논리:
- this paper is about behavior-grounded routing semantics and stage comparability,
- dense soft mixing gives cleaner usage statistics and more stable diagnostics,
- sparse top-k routing is left as a natural extension but is not necessary for validating the current claim.

즉,
"못 해서 안 했다"가 아니라
"이번 논문 목적에 더 적합해서 이렇게 했다"
로 보이게 해야 한다.

[4-4. Appendix B.1의 dataset characterization를 main text 해석과 직접 연결]
Appendix B.1은 이미 dataset별 behavioral regime 설명이 괜찮다.
이걸 4.2 결과 해석과 연결하는 문장만 더 있으면 stronger story가 된다.

예시:
"This pattern is consistent with the dataset characterization in Appendix B.1: datasets with stronger short-horizon or concentrated behavioral signals provide clearer scope-separated routing cues than more stable preference-driven corpora."

5. 정당화 논리: 내 과정과 실험을 어떻게 방어할 것인가
==================================================

[5-1. 왜 sessionization을 유지하는가]
이건 이제 바꾸기보다 정당화하는 편이 낫다.
핵심 논리는 다음이다.
- RouteRec은 session-aware behavioral scope를 전제로 함
- macro / mid / micro의 시간 범위 해석은 sessionization 위에서 더 자연스럽다
- full-history로 되돌리면 오히려 behavior scope 정의가 흐려질 수 있다
- sessionization은 ad hoc tuning trick이 아니라, behavior scope를 operationalize하는 modeling choice라고 설명하면 된다

[5-2. 왜 feature bank가 hand-designed여도 괜찮은가]
방어 포인트:
- 목표는 feature engineering contest가 아니라 routing control interface 설계다
- cue bank는 compact / portable / semantically legible해야 한다
- representative family design + appendix examples + ablation으로 정당화한다
- heavy side-information branch보다 ordinary logs 기반 control signal이 논문 핵심이다

[5-3. 왜 dense mixture를 써도 되나]
방어 포인트:
- 현재 논문은 large-scale sparse MoE efficiency paper가 아니라 behavior-grounded conditional computation paper다
- dense soft mixing은 stage-wise diagnostics와 comparability를 더 안정적으로 준다
- top-k는 extension으로 남겨도 core claim(behavior cue가 routing semantics를 더 낫게 만든다)을 검증하는 데 필수는 아니다

[5-4. 왜 objective에 load balancing이 없나]
방어 포인트:
- 이 논문의 목적은 uniform traffic 자체가 아니라 feature-aligned specialization이다
- consistency + z-loss는 semantic alignment와 numeric stability를 주고,
  explicit balancing은 오히려 behavior-grounded specialization을 약화시킬 수도 있다
- Appendix D에서 optional balance variant를 보이면 rebuttal에 유리하다

(1) BASE Layers: Simplifying Training of Large, Sparse Models
- 저자: Mike Lewis, Shruti Bhosale, Tim Dettmers, Naman Goyal, Luke Zettlemoyer
- venue: ICML 2021 (PMLR 139)
- 링크: https://proceedings.mlr.press/v139/lewis21a.html
- 왜 useful한가:
  - balancing auxiliary loss 없이도 balanced assignment를 논의하는 대표 레퍼런스다.
  - 네 논문에서 "왜 classical load balancing을 final config에서 빼도 되는가" 또는
    "routing design에서 balancing pressure는 선택의 문제"라는 discussion에 붙이기 좋다.
- 논문에 어디 넣으면 좋은가:
  - related work 2.3 또는 3.5 objective discussion
  - Appendix D regularization variants discussion
- 핵심 포인트:
  - 기존 sparse MoE는 balancing loss/heuristics가 필요할 수 있고,
    BASE는 assignment 자체로 균형을 맞춘다.
  - 즉, balancing이 MoE 설계의 핵심 쟁점 중 하나라는 점을 정당화해 준다.

(2) DSelect-k: Differentiable Selection in the Mixture of Experts with Applications to Multi-Task Learning
- 저자: Hussein Hazimeh et al.
- venue: NeurIPS 2021
- 링크: https://proceedings.neurips.cc/paper/2021/hash/f5ac21cd0ef1b88e9848571aeb53551a-Abstract.html
- 왜 useful한가:
  - top-k gate의 비연속성과 optimization difficulty를 짚는다.
  - 네가 dense soft mixing을 채택하고 top-k를 current scope 밖에 둔 이유를 뒷받침할 수 있다.
- 논문에 어디 넣으면 좋은가:
  - 3.4 dense soft mixing rationale
  - future work에서 sparse/top-k extension 언급 시
- 핵심 포인트:
  - sparse top-k gate는 smooth하지 않아 convergence/statistical performance issue가 생길 수 있다고 본다.

(3) Mixture-of-Experts with Expert Choice Routing
- 저자: Yanqi Zhou et al.
- venue: NeurIPS 2022
- 링크: https://arxiv.org/abs/2202.09368
- 왜 useful한가:
  - routing strategy가 expert under-training / over-specialization과 directly 연결된다는 점을 매우 직접적으로 말한다.
  - 네 논문에서 routing semantics와 utilization discussion을 붙이기에 좋다.
- 논문에 어디 넣으면 좋은가:
  - related work 2.3
  - 3.4 마지막 diagnostics comparability discussion
  - Appendix E routing diagnostics interpretation
- 핵심 포인트:
  - poor routing strategy는 imbalance와 under-trained experts를 부를 수 있고,
    routing rule 자체가 quality와 training behavior에 큰 영향을 준다.

(4) ST-MoE: Designing Stable and Transferable Sparse Expert Models
- 저자: Barret Zoph et al.
- 링크: https://arxiv.org/abs/2202.08906
- 왜 useful한가:
  - training stability와 transfer quality를 전면에 놓고 sparse expert design guide를 제시한다.
  - 네 objective / z-loss / stability narrative와 궁합이 좋다.
- 논문에 어디 넣으면 좋은가:
  - 3.5 z-loss / routing stability motivation
  - Appendix D objective variants
  - transfer appendix(G) discussion
- 핵심 포인트:
  - sparse expert models는 training instability와 fine-tuning uncertainty가 큰 이슈라는 framing을 제공한다.

(5) From Sparse to Soft Mixtures of Experts

9. 간단 결론
==================================================

이번 버전은 이미 상당히 좋아졌고,
이제 논문이 약한 부분은 "핵심 아이디어 부족"이 아니라
"선택한 instantiation과 실험 프로토콜을 얼마나 투명하고 정확하게 박아두느냐" 쪽이다.

즉 다음 단계의 목표는
- exact setting을 더 명확히 하고
- appendix 완성도를 끌어올리고
- 몇 개의 citation으로 methodology/justification를 보강하는 것
이다.

이 정도만 더 정리하면,
비판적인 reviewer에게도 "아이디어는 좋은데 아직 미완성"이 아니라
"설계와 실험을 꽤 신중하게 정리한 paper"로 보일 가능성이 높다.
